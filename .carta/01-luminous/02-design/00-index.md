# Design

Product and software design for Luminous. Covers architecture decisions, the concept model, API contracts, MCP integration, and the cactus canvas engine.

**What belongs here:** Design rationale, API contracts, engine documentation, concept inventories — anything that describes *what* the system is and *why* it's shaped that way.

**What doesn't belong here:** Vision/motivation (that's in 01-vision), implementation guides, or operational docs.
